Warsaw Weather API

Querry examples:

Access all data: http://127.0.0.1:5000/weather

Date endpoints:

Filter by temperature:

Access data with temperatures starting with [your_value] -> http://127.0.0.1:5000/weather?temp_min=[your_value]

Access data with temperatures ending with [your_value] -> http://127.0.0.1:5000/weather?temp_max=[your_value]

Example: Return data in temperature range 0-10 deg C -> http://127.0.0.1:5000/weather?temp_min=0&temp_max=10

Filter by date:

Please follow this date format: yyyy-mm-dd

Access data starting with [your_value] -> http://127.0.0.1:5000/weather?date_start=[your_value]

Access data ending with [your_value] -> http://127.0.0.1:5000/weather?date_end=[your_value]

Example: Return data in date range 2015-01-01 to 2020-12-31 -> http://127.0.0.1:5000/weather?date_start=2015-01-01&date_end=2020-12-31

Access data with selected month -> http://127.0.0.1:5000/weather?month=[your_value]

Example: Return data where month is May -> http://127.0.0.1:5000/weather?month=5

Mix the querries:

You can add more filtration to get more detailed output:

Data in 2012-01-01 to 2018-01-01 range for temperatures up to -5 deg C -> http://127.0.0.1:5000/weather?date_start=2012-01-01&date_end=2018-01-01&temp_max=-5
